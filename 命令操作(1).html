2、设置签名  
  作用 ：区分是谁提交的代码
  命令： 
   2.1 项目级别/仓库级别：仅在当前本地库范围内有效
       git config user.name [名称]
       git config user.email [邮箱]

信息存在：./.git/config 文件
内容：
lk@DESKTOP-FQ88HLD MINGW64 /f/gistest (master)
$ cat ./.git/config
[core]

        filemode = false
        bare = false
        logallrefupdates = true
        symlinks = false

        name = lk


 2.2系统用户级别：登录当前操作系统的用户范围
   git --global config user.name [名称]
   git --global config user.email [邮箱]

 2.3级别优先级
      1.就近原则：项目级别优先于系统用户级别，二者都有时采用项目级别 的签名
      2.如果只有系统用户级别的签名，就以系统用户级别的签名为
      3.二者都没有不允许

4、前进后退  相当于 log前面的hash值

4.1 git reset --hard [hash值]
$ git reset --hard 2a03408
HEAD is now at 2a03408 first commit

4.2 使用^符号：只能后退

       git reset --hard HEAD^    一个^代表一个版本
4.3  使用～符号：只能后退
   
       git reset --hard HEAD~n   n代表向前回滚n次
   ps:
--hard  在本地库移动 HEAD 指针 重置工作区  重置暂存区
--soft  仅在本地库移动指针
--mixed  在本地库移动指针 重置暂存区


5、删除文件找回
       就是版本的回滚  git reset --hard [hash值]
